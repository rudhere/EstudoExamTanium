1.1 A Engenharia por trás da Linear Chain Architecture

Os sistemas tradicionais baseiam-se em arquiteturas Hub-and-Spoke (Estrela), que falham em escala massiva (100.000+ computadores) devido à saturação de banda WAN e latência excessiva do banco de dados central.

Fluxo de Dados: Eleição de Liderança e Redução de Banda

Os agentes locais realizam Neighbor Discovery no mesmo domínio de broadcast (Camada 2). O Tanium Client eleito líder (Subnet Leader) atua como ponte para a WAN, distribuindo comandos (Forward Chain) e aggregando respostas (Backward Chain).

Tráfego WAN Tradicional = N * S (Linear com número de agentes)
Tráfego WAN Tanium = C * S (Constante, onde C = número de cadeias lógicas)
1.2 Tanium Client, Server e Fluxo Criptográfico

Como o Tanium opera com privilégios máximos (SYSTEM / root), a segurança baseia-se em criptografia assimétrica de curva elíptica e verificação de hashes.

tanium-init.dat

Contém exclusivamente a chave pública do Tanium Server. Mensagens não assinadas pela chave privada correspondente são descartadas na porta 17472.

client.cfg

Arquivo de parametrização local. Guarda IPs dos servidores, verbosidade de logs (LogVerbosityLevel) e regras de pareamento local.

Zone Server (DMZ)

Proxy reverso TLS seguro intermediando comunicações externas das estações em trânsito pela Internet para o core corporativo interno.

1.3 Conceitos Fundamentais da Console Tanium
Questions (Perguntas)Instruções de busca declarativas. Não interrogam banco central; solicitam que todos os computadores online calculem um script local de forma síncrona.
Sensors (Sensores) & SandboxingScripts estritamente Read-only com timeout padrão de 60 segundos, executados em sandbox restrito temporário.
Actions & PackagesOrdem de modificação (escrita). O agente baixa fatias do pacote via cadeia local, descompacta e executa o script corretor.
Groups & Custom TagsGrupos Dinâmicos atualizam filiações por regras lógicas. Tags são gravadas persistentemente em arquivos locais do agente.
D1: Asking Questions (~22%)

Fórmula do parser em linguagem natural: Get [Sensor] from all machines with [Condition]

  • Get Clause: Define colunas de metadados (sensores) a extrair.
  • From Clause: Define a origem. Padrão = todos os ativos corporativos.
  • With Clause: Define filtros lógicos aplicados nos hosts.
  • Foco TCO: Sensores Parametrizados usam colchetes []. Ex: Get File Exists[C:\temp\patch.log]
D2: Refining Questions (~23%)
Filtros Locais (Browser RAM)

Processamento rápido apenas no navegador, sem reemitir Questions ao servidor.

Filtros no Servidor (Server-side)

Emite nova pergunta com restrição, recalculando dados em tempo real dos endpoints.

Mecanismo Drill Down: Selecionar hosts em tabela existente e submeter nova pergunta focada neles.
D3: Taking Action (~15%)
  • Distribute Over: Randomiza e dilui downloads ao longo do tempo para evitar sobrecarga em storages.
  • Action Statuses: Scheduled -> Downloading -> Running -> Completed (Exit 0) ou Failed (Exit != 0)
  • One-time vs Scheduled: Agendadas recorrentes usam Saved Questions dinâmicas para conformidade contínua.
  • Action Lock: Trava local no cliente impedindo execução forçada em hosts críticos.
D4: Navigation and Modules (~23%)
Interact:Cockpit central focado em perguntas em linguagem natural e ações ad-hoc.
Comply:Auditoria e conformidade com CVEs públicas e guias CIS de hardening.
Patch:Esteiras e aplicação controlada de atualizações em janelas de manutenção.
Threat Response:Forense em tempo real, Kernel Recorders e isolamento emergencial de endpoints.
D5: Reporting & Export (~17%)
  • Interact UI Export: CSV ou JSON direto do navegador.
  • Tanium Connect: Barramento para SIEMs — Splunk HEC, Azure Sentinel, Syslog RFC 5424, AWS S3, ServiceNow.
  • Delta Export: Compara varreduras atuais com cache e envia apenas hosts com mudanças, economizando banda WAN.

Parte 5 — Questões Resolvidas de Exemplo
Fácil

Qual porta padrão os Tanium Clients usam para conexões de cadeia linear na LAN?

Gabarito: TCP 17472. Porta exclusiva de barramento para mensagens de controle, vizinhança e cadeia linear.

Médio

O que ocorre se um endpoint na Linear Chain sofrer desligamento repentino?

Gabarito: Vizinhos detectam a perda e reorganizam o anel dinamicamente via Neighbor Timeout e Neighbor Discovery.

Difícil

Qual a vantagem de configurar Delta Export no Tanium Connect?

Gabarito: Transmitir apenas alterações em relação à última varredura, poupando banda WAN e impedindo congestionamento no SIEM.

Parte 6 — Pegadinhas Comuns
Pegadinha 1: Sensores não modificam nada

A prova sugere que um "Sensor" pode encerrar serviços. Falso! Sensores são Read-only. Modificações exigem Actions com Packages.

Pegadinha 2: Colchetes vs Parênteses

Sensores Parametrizados: colchetes []. Agrupamentos lógicos: parênteses ().

Pegadinha 3: Local vs Server Filters

Filtros locais no browser processam em RAM sem retransmitir ao servidor.

Pegadinha 4: TCP vs UDP

Descobrimentos usam pings, mas downloads de payloads pesados são estritamente TCP 17472.

Parte 7 — Estratégia de Prova

Tempo: 105 minutos para ~60 questões.

1. Triage Reverso: Resolva primeiro questões teóricas rápidas (1 min cada). Deixe cenários complexos para a segunda metade.

2. Decifre Queries por Blocos: Mentalmente divida em Get Clause e With Clause.

3. Descarte o Impossível: Elimine alternativas que aleguem que o Tanium Server trava quando endpoints ficam offline.
